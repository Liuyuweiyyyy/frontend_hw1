# 社交平台建設計畫

## 🏗️ 這個專案是什麼？

就像在學校做一個「大大布告欄」！
- 同學可以貼便利貼（發布貼文）
- 別人可以貼愛心貼紙（按讚）
- 可以在下面寫小紙條（留言）
- 每個人有自己的座位放資料（個人檔案）

---

## 📋 五個階段詳細說明

### 第一階段：顯示假資料的貼文列表

#### 👩‍🏫 老師解釋
我們要先「假的」放一些貼文，這樣才能看到畫面。就像先在布告欄貼幾張假的便利貼，看看好不好看！

#### 🛠️ 要做的事情
1. 建立 `src/data/mock.js` - 假資料小幫手（放測試資料）
2. 建立 `src/components/PostCard.vue` - 貼文卡片小幫手（顯示單則貼文）
3. 建立 `src/components/PostList.vue` - 貼文列表小幫手（排好所有貼文）

#### 📦 資料是怎麼「出現」的？

```
📦 假資料小幫手（包裹工廠）
    │
    ▼ 「寄出去」
📋 貼文列表小幫手（郵局）
    │
    ▼ 「分送」給大家
📄 貼文卡片小幫手（便利貼）← 終於看到內容了！
```

**經過的關卡**：
- 第一關：假資料小幫手「輸出」資料（把包裹拿出來）
- 第二關：貼文列表小幫手「處理」一下（整理一下）
- 第三關：貼文卡片小幫手「變成畫面」（把資料變成便利貼）

---

### 第二階段：加入導航列

#### 👩‍🏫 老師解釋
我們需要一個「選單」，讓大家可以選擇要看「首頁」還是「我的檔案」。就像教室裡有兩個門，一個去「動態牆」，一個去「個人檔案」！

#### 🛠️ 要做的事情
1. 建立 `src/components/NavBar.vue` - 導航列小幫手（最上面的選單）
2. 建立「頁面盒子」來記錄目前在哪一頁
3. 讓點擊可以切換畫面

#### 🔀 切換頁面是怎麼做到的？

就像選舉投票！

```
👆 點擊「個人檔案」
    │
    ▼ 觸發「切換開關」
📦 某個「盒子」的值從「否」變成「是」
    │
    ▼ 監視器自動發現變化
🎭 畫面「換表演」：從動態牆變成個人檔案
```

**經過的關卡**：
- 第一關：點擊按鈕（啟動開關）
- 第二關：改變「某個盒子」的值（選項改變）
- 第三關：畫面自動更新（監視器發現變化）

---

### 第三階段：發布新貼文

#### 👩‍🏫 老師解釋
讓大家可以寫新的便利貼！就像去超商買東西，投入硬幣、選好商品、掉出來！

#### 🛠️ 要做的事情
1. 建立 `src/components/CreatePost.vue` - 發布小幫手（輸入框 + 按鈕）
2. 按下發布時，把新資料「推進」貼文清單的盒子

#### ✏️ 發布新貼文是怎麼做到的？

```
✏️ 在框框裡打字（輸入資料）
    │
    ▼ 【第一關：收集資料】
📝 把文字「抓」進一個盒子裡
    │
    ▼ 【第二關：啟動開關】
👆 按下「發布」按鈕
    │
    ▼ 【第三關：更新清單】
📋 把新資料「推進」貼文盒子（放在最前面）
    │
    ▼ 【監視器發現變化】
🔄 監視器說：「哎呀！資料變了，要重新畫畫！）
    │
    ▼ 【第五關：畫面更新】
🖥️ 新的便利貼出現在最上方！
```

**經過的關卡**：
- 第一關：在輸入框打字（收集資料）
- 第二關：按發布按鈕（啟動開關）
- 第三關：把資料「放進」清單（更新資料）
- 第四關：畫面自動刷新（監視器發現變化）

---

##### 🚪 每一關的詳細說明

**【第一關】資料收集**

在輸入框打字時，發生了什麼？

就像寫作業時，老師發給你一張「收集紙」：
```
┌─────────────────────────┐
│  貼文內容收集紙          │
│  ┌───────────────────┐  │
│  │                   │  │
│  │   在這裡打字       │  │  ← 輸入框
│  │                   │  │
│  └───────────────────┘  │
└─────────────────────────┘
```

每打一個字，這張紙就會**自動記錄**下來，不需要另外按儲存。

---

**【第二關】啟動開關**

按下「發布」按鈕時，發生了什麼？

就像按下一個「啟動開關」：
```
┌─────────────────────────┐
│  [ 按我發布！ ]  🔘     │  ← 按鈕
└─────────────────────────┘
          │
          ▼ 按下後啟動
    ┌─────────────┐
    │ 發布事件！   │  → 叫出「處理小幫手」
    └─────────────┘
```

這個「處理小幫手」會做兩件事：
1. 把「收集紙」上的文字整理好
2. 叫「貼文盒子」收下這份新資料

---

**【第三關】更新清單**

新資料是如何放進去的？

就像把新玩具放進玩具箱：
```
玩具箱（貼文清單）：[🧸, 🚗, 🎮, ...]
                          │
                          ▼ 加入新成員
                          
玩具箱（貼文清單）：[✨新貼文, 🧸, 🚗, 🎮, ...]
```

我們用一個特別的方法：叫做「從前面放」
- 意思：「從前面放進去」
- 這樣新貼文會出現在**最上面**

---

**【第四關】監視器發現變化**

監視器是怎麼知道資料變了？

就像有個「監視器」一直看著玩具箱：
```
監視器 👀 看著玩具箱
        │
        ▼ 「有新玩具進來了！）
        │
    🔔 按鈴通知：「要重新整理畫面了！）
```

這個監視器會：
- 一直注意「盒子」裡的內容
- 一發現變化，就馬上說：「要重新整理畫面了！」

---

**【第五關】畫面更新**

畫面是怎麼變化的？

就像老師重新畫一次黑板：
```
┌─────────────────────────┐
│  舊的黑板（舊畫面）       │
│  ──────────────────     │
│  🚗 貼文2               │
│  🧸 貼文3               │
└─────────────────────────┘

        ▼ 監視器重新畫

┌─────────────────────────┐
│  新的黑板（新畫面）       │
│  ──────────────────     │
│  ✨ 貼文1（新的！）      │  ← 出現在最上面
│  🚗 貼文2               │
│  🧸 貼文3               │
└─────────────────────────┘
```

### 第四階段：按讚功能

#### 👩‍🏫 老師解釋
讓大家可以點愛心！就像開關電燈，一按就亮、再按就暗！

#### 🛠️ 要做的事情
1. 在每則貼文裡放一個「讚了嗎」盒子（有讚/沒讚）
2. 點擊時切換盒子的值
3. 根據盒子的值，顯示不同顏色的愛心

#### ❤️ 按讚是怎麼做到的？

```
👆 點擊愛心
    │
    ▼ 觸發「切換開關」
📦 貼文的「讚數盒子」+1，「有沒有讚」反轉
    │
    ▼ 監視器自動發現變化
💖 愛心變色、數字增加
```

**經過的關卡**：
- 第一關：點擊愛心（啟動開關）
- 第二關：更新「讚數盒子」和「有沒有讚」（開/關）
- 第三關：畫面自動更新（愛心變色、數字變）

---

### 第五階段：個人檔案

#### 👩‍🏫 老師解釋
每個人有自己的「專屬座位」！就像身份證一樣，有照片、名字、自己寫過的便利貼！

#### 🛠️ 要做的事情
1. 建立 `src/components/UserProfile.vue` - 個人檔案小幫手
2. 把目前用戶的資料獨立出來
3. 篩選顯示「這個用戶」發過的貼文
4. 顯示頭像、名字、自我介紹

#### 👤 個人檔案是怎麼出現的？

```
👆 點擊導航列的「個人檔案」
    │
    ▼ 切換到「檔案頁面」
📦 從假資料小幫手找出「我的」貼文
    │
    ▼ 把資料傳給檔案小幫手
👤 我的專屬頁面出現了！
```

**經過的關卡**：
- 第一關：點擊導航（啟動開關）
- 第二關：改變「目前頁面」這個盒子（選項改變）
- 第三關：顯示檔案小幫手（畫面出現）
- 第四關：找出我的貼文（挑出屬於我的資料）


## 🎯 驗收標準

每個階段完成後，要能：
1. **第一階段**：看到假資料變成的便利貼列表
2. **第二階段**：點擊導航可以切換首頁/個人檔案
3. **第三階段**：寫完文字、按發布，新貼文出現在最上面
4. **第四階段**：點擊愛心，愛心變色、數字增加
5. **第五階段**：看到自己的頭像、名字、所有發過的貼文
